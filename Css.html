Chapter 01

What is CSS?

CSS (Cascading Style Sheets) is the presentation layer of the web. It controls colours, fonts, spacing, layout, animations, and even 3D effects. Without CSS, the web would be plain text on a white background. The word "Cascading" refers to the priority system that determines which style wins when multiple rules target the same element.

💡 Why learn CSS deeply? It’s the only way to build beautiful, responsive, and professional websites. Combined with HTML and JavaScript, it's the foundation of front‑end development.

}

Chapter 02

Three Ways to Add CSS

You can style HTML using inline (style attribute), internal (<style> tag), or external (separate .css file). External is the professional standard.

🔹 Inline CSS

This paragraph uses inline CSS (style attribute).

<p style="color: blue; font-weight: bold;">Text</p>

🔹 Internal CSS

Styled via internal <style> tag.

<style>
  .internal-example { color: green; }
</style>

🔹 External CSS (Best Practice)

<link rel="stylesheet" href="styles.css">
✅ External CSS: separate file, cacheable, reusable across many pages → professional workflow.
Chapter 03

Selectors & Box Model

Selectors target HTML elements: element (p), class (.card), ID (#header).
Box Model: margin (outside), border, padding (inside), content. Always set box-sizing: border-box;.
Content area – margin, border, padding visible.
* { box-sizing: border-box; }
.card {
  margin: 20px;
  border: 3px solid blue;
  padding: 20px;
}
Chapter 04

Specificity & Cascade

The browser decides which CSS rule wins by specificity score: inline (1000) > ID (100) > class (10) > element (1). Later rules also override if equal. Never abuse !important.

Click button to see specificity overrides.

#id { color: red; }   /* specificity 100 */
.class { color: blue; }  /* specificity 10 */
Chapter 05

2D Transforms (rotate, scale, translate, skew)

2D transforms allow you to move, rotate, scale, and skew elements. They are GPU‑accelerated and perfect for interactive UIs. Properties: transform: rotate(deg) scale() translate(x, y) skew(xdeg, ydeg);
Use them on :hover or with JavaScript for dynamic effects. The interactive panel below lets you combine all transforms in real time.

/* Basic 2D transforms */
.rotate:hover { transform: rotate(15deg); }
.scale:hover { transform: scale(1.2); }
.translate:hover { transform: translate(20px, 20px); }
.skew:hover { transform: skew(10deg, 5deg); }

/* Combined transforms (order matters!) */
.combined {
  transform: rotate(30deg) scale(1.5) translate(20px, 20px) skew(5deg);
}
✨ Order matters! Transform functions are applied left to right. For example, translate(50px) rotate(45deg) is different from rotate(45deg) translate(50px).
Chapter 06

3D Transforms & Perspective

3D transforms add depth by rotating elements around X, Y, or Z axes. You need perspective on the parent container to create the 3D effect. transform-style: preserve-3d allows children to be positioned in 3D space.
Use rotateX() (tilt forward/back), rotateY() (turn left/right), and rotateZ() (spin). The interactive sliders below let you control the 3D card in real time.

🎯 Perspective defines how strong the 3D effect is (lower value = more dramatic). transform-style: preserve-3d is required for nested 3D elements.
Chapter 07

Filters & Keyframe Animations

Filters (blur, grayscale, sepia, brightness, contrast, hue-rotate) apply visual effects similar to Photoshop.
Keyframe animations let you create complex, repeating movements using @keyframes.
Bounce
Spin
/* Filters */
img {
  filter: blur(2px) grayscale(50%) sepia(20%) brightness(1.2);
}

/* Keyframe animation */
@keyframes bounce {
  0%, 100% { transform: translateY(0); }
  50% { transform: translateY(-20px); }
}
.bounce {
  animation: bounce 1s infinite ease;
}

Chapter 09

Responsive Design & Media Queries

Media queries apply CSS only when certain conditions are met (e.g., screen width). Mobile‑first means write default styles for small screens, then use min-width for larger devices.
Resize browser → my background changes! (blue: <600px, green: 600-1023px, purple: ≥1024px)
/* Mobile first: default for small screens */
.box { background: blue; }
@media (min-width: 600px) {
  .box { background: green; }
}
@media (min-width: 1024px) {
  .box { background: purple; }
}
Chapter 10

CSS Variables & Pseudo-classes

CSS Variables (custom properties) store values for reuse. Pseudo-classes like :hover, :first-child, :nth-child() style elements based on state or position.
  • ✅ First child (green)
  • 🔹 Second child
  • 🔹 Third child

  background: var(--primary);
}
button:hover {
  transform: scale(1.05);
}
li:first-child { color: green; }
Chapter 11

CSS Best Practices

  • ✅ Use external CSS for real projects.
  • ✅ Prefer classes over IDs for styling (lower specificity, more reusable).
  • ✅ Animate transform and opacity (GPU accelerated, smoother).
  • ✅ Always set box-sizing: border-box; globally.
  • ✅ Write mobile‑first media queries.
  • ✅ Avoid !important except for utility overrides.
  • ✅ Use CSS variables for consistent theming.
  • ✅ Test with browser DevTools (F12).
🔥 Pro tip: Use will-change: transform for elements you plan to animate frequently to improve performance.
