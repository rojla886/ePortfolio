🎨 1. What is CSS?

CSS (Cascading Style Sheets) is the language that makes websites beautiful. If HTML is the skeleton of a webpage, CSS is the skin, clothes, and makeup – it controls colours, fonts, spacing, layout, animations, and even 3D effects. Without CSS, every website would look like a plain black‑and‑white document from the 1990s.

The word "Cascading" means that styles can flow (or cascade) from parent elements to children, and that multiple style rules can apply to the same element – with a well‑defined order of priority (specificity). "Style Sheets" refer to the fact that you write rules that define the visual presentation of HTML elements.

CSS was first proposed in 1994 by Håkon Wium Lie and became a web standard in 1996. Since then, it has evolved through versions CSS1, CSS2, CSS3 (which introduced modules like transforms, animations, grids, and flexbox), and today we use CSS3 with continuous updates.
Without CSS: Plain text, no styling, ugly defaults.
With CSS: Colours, bold, spacing, rounded corners, shadows, and smooth animations.
This box uses CSS – gradient background, hover effect (scale), rounded corners, and a soft shadow. 
/* CSS syntax: selector { property: value; } */
p {
  color: navy;           /* text colour */
  font-size: 18px;      /* font size */
  background: #f0f0f0;  /* background colour */
  border-radius: 8px;   /* rounded corners */
  transition: 0.3s;     /* smooth hover effect */
}
p:hover {
  transform: scale(1.02);
}
💡 CSS follows a simple pattern: selector { property: value; } – choose an element, then decide how it should look. The cascade means later rules can override earlier ones, and children inherit some properties from their parents.

📌 2. Three Ways to Add CSS (Inline, Internal, External)

CSS can be added to HTML in three different ways. Each has its purpose. Think of it like decorating a house: you can paint a single room (inline), create a style guide for one house (internal), or have a central design manual for all houses (external). Understanding these methods is fundamental because they affect maintainability, reusability, and page loading speed.

2.1 Inline CSS (inside an HTML element)

How it works: You write CSS directly inside an HTML tag using the style attribute. This affects only that one element. It’s like putting a sticky note on a single item – quick but not reusable.

When to use: Only for quick testing, one‑off overrides, or when you need to apply a style that shouldn't affect anything else. In real projects, avoid inline CSS because it mixes content with presentation and is hard to maintain.

This paragraph is styled with inline CSS – look at the code.

This div also uses inline CSS – background, padding, border, etc. 
<p style="color: blue; font-weight: bold;">This is inline CSS</p>
<div style="background: yellow; padding: 0.5rem;">Styled div</div>
⚠️ Use inline CSS only for one‑off overrides. It mixes content with styling and is hard to maintain in large projects. It also overrides all other CSS because of high specificity.

2.2 Internal CSS (inside <style> tag in <head>)

How it works: You write CSS inside a <style> tag in the HTML <head>. This applies to the entire HTML page. It’s like having a style sheet for one room – good for single‑page demos or when you don't want an extra file.

When to use: Single‑page websites, quick prototypes, or when you want to demonstrate CSS without needing external files. However, if you have multiple pages, you'll repeat the same CSS, which is inefficient.

This paragraph uses an internal CSS class – defined in the <style> tag of this page.

Another internal CSS styled box.
<style>
  .internal-demo { color: green; font-weight: bold; }
  .internal-demo-box { background: lightblue; padding: 0.5rem; }
</style>
✅ Internal CSS is great for single‑page prototypes. For multi‑page sites, external CSS is better (reusable and cacheable).

2.3 External CSS (separate .css file)

How it works: You store all CSS in a separate .css file and link it using <link> in the HTML <head>. This is the best practice for real websites because one CSS file can style many HTML pages, and browsers can cache it for faster loading.

When to use: All professional projects, multi‑page websites, and team development. It separates concerns (HTML for structure, CSS for styling) and makes maintenance effortless – change one file, and every page updates.

This entire page is styled with a mix of internal and external concepts. In a real project, all styles would be in an external file like styles.css.

For example, the global styles for this page (like the navigation bar, card designs, etc.) are written in a <style> block – but imagine they are in style.css and linked.

<!-- In HTML head -->
<link rel="stylesheet" href="styles.css">

/* styles.css (separate file) */
body {
  font-family: Arial;
  background: #f0f0f0;
}
button {
  background: blue;
  color: white;
}
✅ External CSS keeps HTML clean, allows browser caching (faster subsequent page loads), and is reusable across many pages – essential for professional work.

📚 3. CSS Fundamentals – Selectors, Colours, Box Model

The core of CSS: You select an HTML element, then you apply properties. Let’s break down the most important concepts that every developer must know.

🎯 Selectors – How to target HTML elements

Selectors are patterns used to select the element(s) you want to style. There are several types:
  • Element selector – selects all elements of a given type: p { }
  • Class selector – selects elements with a specific class attribute: .highlight { }
  • ID selector – selects a unique element with a specific id: #header { }
  • Descendant selector – selects elements inside other elements: div span { }
  • Group selector – applies same styles to multiple selectors: h1, h2, h3 { }
  • Attribute selector – selects elements based on an attribute: input[type="text"] { }

This paragraph uses a class selector (.selector-demo)

This div uses an ID selector (#unique-demo)
Descendant selector: this span inside a div is targeted by "div span"
/* Element selector – targets all <p> */
p { line-height: 1.5; }

/* Class selector – targets elements with class="highlight" */
.highlight { background: yellow; }

/* ID selector – targets element with id="header" */
#header { font-size: 2rem; }

/* Descendant selector – targets <span> inside <div> */
div span { color: blue; }

/* Group selector */
h1, h2, h3 { font-family: sans-serif; }

🎨 Colours & Backgrounds

CSS supports many colour formats:
  • Hex – #RRGGBB (e.g., #ff5733)
  • RGB – rgb(red, green, blue) (e.g., rgb(255,87,51))
  • HSL – hsl(hue, saturation, lightness) (e.g., hsl(9, 100%, 60%))
  • Named colours – red, blue, tomato, etc.
Backgrounds can be solid colours, gradients (linear or radial), or images.
Hex colour (#ef4444) – red
RGB colour (59,130,246) – blue
Linear gradient – smooth transition between colours
/* Color formats */
color: #ff5733;        /* Hex */
color: rgb(255, 87, 51); /* RGB */
color: hsl(9, 100%, 60%); /* HSL */
background: linear-gradient(to right, red, yellow);

📦 Box Model (Margin, Border, Padding, Content)

Every element in CSS is a rectangular box. The box model consists of:
  • Content – the actual text or image
  • Padding – space between content and border (inside the element)
  • Border – a line around the padding (or content if no padding)
  • Margin – space outside the border (between this element and others)
By default, width/height only apply to content. To make layouts predictable, use box-sizing: border-box; – then width includes padding and border.
Content area
margin (space outside) → border → padding (space inside) → content 
/* Box model */
div {
  margin: 20px;      /* space outside the border */
  border: 3px solid blue;
  padding: 20px;     /* space inside the border, around content */
  width: 200px;      /* content width */
}

/* Best practice: include padding/border in width */
* {
  box-sizing: border-box;
}
⭐ Always add box-sizing: border-box; globally to make layouts predictable – it includes padding and border inside the element's total width/height.

⚖️ 4. Specificity, Cascade & Inheritance

Specificity decides which CSS rule wins when multiple rules target the same element. The browser calculates a score: inline styles (1000) > ID selectors (100) > class/attribute/pseudo-class selectors (10) > element selectors (1). If two rules have the same specificity, the later one (cascade) wins. Inheritance means some properties (like colour, font) are passed from parent to child elements automatically.

💡 Use classes instead of IDs for styling – lower specificity makes your code more reusable and easier to maintain. Avoid !important unless absolutely necessary (e.g., overriding third‑party styles).

📦 5. CSS Variables (Custom Properties)

What they are: CSS variables store values (colours, sizes, etc.) in one place, so you can reuse them throughout your stylesheet. Define them inside :root (global) or any selector. Then use var(--name) to apply them. Changing the variable once updates everywhere – perfect for theming (light/dark modes), consistent design systems, and easier maintenance.

Variables are also inherited (child elements can use parent variables) and can be updated with JavaScript.
Background set via CSS variable – change one line, update everywhere.
:root {
  --primary-color: #2563eb;
  --spacing: 1rem;
}
button {
  background: var(--primary-color);
  padding: var(--spacing);
}

🎭 6. Pseudo‑classes & Pseudo‑elements

Pseudo‑classes (like :hover, :focus, :first-child) style elements based on state or position. Pseudo‑elements (like ::first-line, ::before, ::after) style specific parts of an element.

Examples: :hover changes style when mouse is over, :first-child targets the first child element, ::first-line styles only the first line of text, ::before inserts content before an element (often used for icons).

This paragraph uses ::first-line pseudo‑element – the first line is bold and blue. Resize the browser to see it change.

  • First child (styled differently)
  • Second child
  • Third child
/* Pseudo-class – styles based on user interaction */
button:hover { background: darkblue; }

/* Pseudo-element – styles part of an element */
p::first-line { font-weight: bold; }

/* Structural pseudo-class */
li:first-child { color: green; }

📱 7. Responsive Design – Media Queries

Media queries allow you to apply different CSS rules based on screen size, orientation, or device type. This is how websites work on phones, tablets, and desktops. The concept is "mobile‑first" – write styles for small screens first, then use min-width to add enhancements for larger screens.

Resize your browser window – the box below changes colour when the width is less than 600px (phone) and again when it's larger (tablet/desktop).
Resize me (width < 600px turns red, ≥600px green, ≥1024px purple)
/* Mobile first: default styles for small screens (under 600px) */
.responsive-box { background: blue; }

/* Tablet and up (min-width 600px) */
@media (min-width: 600px) {
  .responsive-box { background: green; }
}

/* Desktop (min-width 1024px) */
@media (min-width: 1024px) {
  .responsive-box { background: purple; }
}
💡 Always use relative units (rem, %, vw) and media queries to make your designs mobile‑friendly. Test on actual devices or browser DevTools.

🎨 10. CSS Filters & Keyframe Animations

Filters (blur, grayscale, sepia, brightness, contrast, hue-rotate, etc.) apply visual effects directly to images or elements. backdrop-filter affects the area behind an element (creating glassmorphism effects).

Keyframe animations allow complex, repeated movements – you define keyframes (states) with @keyframes and then animate between them. This is more powerful than transitions because you can have multiple steps and loops.

/* Filters */
img {
  filter: blur(2px) grayscale(50%) sepia(20%);
}

/* Keyframe animation */
@keyframes bounce {
  0%, 100% { transform: translateY(0); }
  50% { transform: translateY(-20px); }
}
.bounce {
  animation: bounce 1s infinite ease;
}

📐 11. Layout: Flexbox & CSS Grid

Flexbox is for one‑dimensional layouts – arranging items in a row or column, with powerful alignment options (justify-content, align-items). It's perfect for navigation bars, card layouts, and centering.

CSS Grid is for two‑dimensional layouts – creating complex grids of rows and columns. You can define both row and column tracks, place items precisely, and create responsive layouts without media queries (using auto-fit/minmax).

Both are essential for modern responsive web design and are supported in all modern browsers.

✅ 12. CSS Best Practices

Following these best practices will make your CSS maintainable, performant, and professional.
  • ✔ Use external CSS for real projects (separate concerns – HTML for structure, CSS for styling).
  • ✔ Prefer classes over IDs for styling (lower specificity, more reusable, easier to override).
  • ✔ Use transform and opacity for smooth animations (GPU accelerated, better performance). Avoid animating width/height/top/left.
  • ✔ Always include box-sizing: border-box; globally to make layouts predictable.
  • ✔ Write responsive styles with @media queries using a mobile‑first approach (default styles for small screens, then add with min-width).
  • ✔ Use CSS variables for consistent theming and easy updates.
  • ✔ Comment your code and keep selectors shallow (avoid deep nesting like .nav .menu ul li a).
  • ✔ Avoid !important unless overriding third‑party styles – it breaks the natural cascade.
  • ✔ Test your website on multiple browsers and devices (Chrome DevTools has device emulation).
  • ✔ Use browser DevTools (F12) to inspect and experiment with CSS live – it's the best learning tool.
