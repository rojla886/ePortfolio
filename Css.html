📌 1. Three Ways to Write CSS (Inline, Internal, External)

CSS can be added to HTML in three different ways. Each has its own purpose and use case. Let's explore them with live examples.

1.1 Inline CSS (inside an HTML element)

Inline CSS is written directly inside an HTML tag using the style attribute. It affects only that single element.

This paragraph is styled with inline CSS (look at the code).

This div also uses inline CSS – background, padding, border, etc. 
<p style="color: blue; font-weight: bold;">Text</p>
✅ Use inline CSS only for very specific, one‑off overrides. It’s not recommended for large projects because it mixes content with styling and is hard to maintain.

1.2 Internal CSS (inside <style> tag in <head>)

Internal CSS is written inside a <style> tag in the <head> section. It applies to the entire HTML page.

This paragraph uses an internal CSS class (look at the style block in the head).

Another internal CSS styled box
<style>
    .internal-demo {
        color: green;
        font-weight: bold;
    }
</style>
✅ Internal CSS is good for single‑page websites or quick prototypes. For multi‑page sites, external CSS is better.

1.3 External CSS (separate .css file)

External CSS is stored in a separate .css file and linked using <link>. This is the best practice for real websites – one CSS file can style many HTML pages.

This entire page is styled with external CSS (the styles you see in the <style> tag could be moved to a .css file).

For demonstration, imagine a file called styles.css: 

<!-- In HTML head -->
<link rel="stylesheet" href="styles.css">

/* styles.css */
body {
    font-family: Arial;
    background: #f0f0f0;
}
✅ External CSS keeps your HTML clean, makes styling reusable, and allows browser caching for faster loading.

📊 Comparison Table: Inline vs Internal vs External

Feature Inline CSS Internal CSS External CSS
Where it's written Inside style attribute of an element Inside <style> tag in <head> Separate .css file linked via <link>
Applies to Only one element Whole HTML page Any page that links to it
Best for Quick, one‑off overrides Single‑page demos Professional, multi‑page websites
Maintainability Hard (mixes content & style) Medium Easy (centralised)
Performance Slow if overused Good for one page Best (cached by browser)
💡 For your portfolio, you will mostly use external CSS or internal CSS for single‑page demos. Inline is rarely used in real projects.

📚 2. CSS Fundamentals

What is CSS? CSS (Cascading Style Sheets) makes websites beautiful. It controls colours, fonts, spacing, and even 3D effects. 
/* Syntax: selector { property: value; } */

    color: blue;
    font-size: 18px;
}

✅ 7. Best Practices

  • ✔ Use external CSS for multi‑page websites.
  • ✔ Use transform and opacity for smooth animations (GPU accelerated).
  • ✔ Always include box-sizing: border-box; globally.
  • ✔ Use meaningful class names (e.g., .nav-bar).
  • ✔ Test responsiveness with media queries.
