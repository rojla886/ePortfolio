Chapter 01: The Foundation

What is HTML? (Detailed Definition)

HTML is the standard markup language for documents designed to be displayed in a web browser. It can be assisted by technologies such as Cascading Style Sheets (CSS) and scripting languages such as JavaScript. HTML elements are the building blocks of HTML pages. With HTML constructs, images and other objects, such as interactive forms, may be embedded into the rendered page.

Example 
<p>Hello World</p>
What this code does:
  • The <p> (Paragraph) tag defines a block-level element for text.
  • The browser automatically adds some white space (margin) before and after each paragraph.
  • The closing </p> tells the browser the element ends here.
Chapter 02: Document Anatomy

The Standard HTML Skeleton

A properly formatted HTML document must include specific metadata and wrapping tags to ensure compatibility across all browsers (Chrome, Safari, Firefox, Edge).

Skeleton 
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Professional Website</title>
</head>
<body>
    <!-- Visible content goes here -->
</body>
</html>
Component Breakdown:
!DOCTYPEForces the browser to render in "Standards Mode".
html lang="en"Defines English as the language for accessibility and search engines.
meta charset="UTF-8"Supports almost all characters and symbols in the world.
bodyThe container for all visible elements (text, images, links).
Chapter 03: Typography

Headings and Text Hierarchies

Headings are defined with the <h1> to <h6> tags. <h1> defines the most important heading. <h6> defines the least important heading. Search engines use the headings to index the structure and content of your web pages.

<h1>Heading 1</h1>
<h2>Heading 2</h2>
<p>This is <strong>important</strong>.</p>

Heading 1

Heading 2

This is important.

Technical Details:

The <strong> tag is semantic, meaning it tells the browser the text has "strong importance." Visually it looks bold, but for a screen reader, it signifies emphasis.

Chapter 04: Grouping Content

Ordered and Unordered Lists

HTML offers three types of lists: Unordered (bullet points), Ordered (numbers/letters), and Description (terms/definitions).

<ul> <!-- Unordered -->
  <li>Item A</li>
</ul>

<ol> <!-- Ordered -->
  <li>Step 1</li>
</ol>
What this code does:

The <li> (List Item) tag must always be wrapped in either a <ul> or <ol>. Browsers add indentation and markers (bullets or numbers) by default.

Chapter 05: Navigation

The Power of Anchor Links

Links are the most important part of the web. They connect pages and documents.

<a href="https://google.com" target="_blank">Visit Google</a>
Execution Breakdown:
  • href: Specifies the URL of the page the link goes to.
  • target="_blank": Instructs the browser to open the link in a new tab.
  • Without an href, the anchor tag is just a placeholder and cannot be clicked.
Chapter 06: Visuals

Multimedia: Images, Audio, Video

HTML5 allows for rich media embedding without third-party plugins like Flash.

<img src="image.jpg" alt="Description" width="500">

<video controls>
  <source src="movie.mp4" type="video/mp4">
</video>
Component Breakdown:
  • src: The path to the file.
  • alt: Crucial for SEO and Screen Readers. If the image fails to load, this text is displayed.
  • controls: Adds the play, pause, and volume interface to the video/audio player.
Chapter 07: Data Structures

Tabular Data

Tables are used to display data in rows and columns. They should not be used for creating website layouts.

TagDescription
<tr>Table Row
<td>Table Data (Cell)
What this code does:

The <thead> and <tbody> tags are used to group header and body content in an HTML table, which is very helpful for styling and accessibility.

Chapter 08: Interaction

User Input Forms

Forms are used to collect user input, which is then sent to a server for processing.

<form action="/submit" method="POST">
  <input type="email" name="email" required>
  <button type="submit">Send</button>
</form>
Execution Breakdown:
  • action: The URL where the form data will be sent.
  • method="POST": Sends the data securely in the body of the HTTP request.
  • name: Very important! Without a name attribute, the server will not receive the input value.
Chapter 09: Modern Layouts

Semantic HTML5

A semantic element clearly describes its meaning to both the browser and the developer.

Key Semantic Tags:
<header>
<nav>
<main>
<article>
<footer>
Why use this?

Google's crawlers use these tags to determine what part of your page is the navigation and what part is the primary content, significantly boosting your SEO ranking.

Chapter 11: Characters

HTML Character Entities

Reserved characters in HTML must be replaced with character entities.

Common Entities:
  • &lt; displays as <
  • &gt; displays as >
  • &copy; displays as ©
  • &nbsp; creates a non-breaking space.
Chapter 12: Embedding

Iframes (Internal Browsing)

An inline frame is used to embed another document within the current HTML document.

<iframe src="page.html" title="Description"></iframe>
Security Note:

Always include a title for accessibility. Many modern sites block being put in iframes for security (Clickjacking protection).

Chapter 13: SEO

The Hidden Logic: Meta Tags

Meta tags provide metadata about the HTML document. Metadata will not be displayed on the page, but will be machine parsable.

<meta name="viewport" content="width=device-width">
<meta name="description" content="Learn HTML free">
What this code does:

The viewport meta tag is essential for Responsive Design. Without it, mobile browsers will show your desktop site at a tiny, zoomed-out scale.

Chapter 15: Security

Built-in Form Validation

HTML5 can automatically check if a user entered the correct data before sending it to the server.

<input type="number" min="18" max="99" required>
Logic Breakdown:

If the user enters "15" or leaves the field empty, the browser will block the form submission and show a popup warning automatically.
