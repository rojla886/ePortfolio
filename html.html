Chapter 01

What is HTML?

HTML (**HyperText Markup Language**) is the most basic building block of the Web. It defines the meaning and structure of web content. It's not a programming language, but a markup language that uses tags to annotate text, images, and other content for display in a Web browser.

Why learn it? Every single website, from Google to Facebook, is built on a foundation of HTML. Without it, the web would be plain text.

✨ HTML tags usually come in pairs: an opening tag like <p> and a closing tag </p>. The content lives between them. Browsers interpret these tags to create headings, paragraphs, links, forms, and rich media.

<p>This is a paragraph tag.</p>
Chapter 02

The Standard Skeleton

Every HTML file must follow this exact document structure to be rendered correctly by browsers.

📄 The <!DOCTYPE html> declaration ensures the browser uses modern standards mode. The <head> holds metadata, and everything visible goes inside the <body>.

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Page Title</title>
</head>
<body>
    <!-- Content goes here -->
</body>
</html>
Chapter 03

Text & Headings

HTML offers 6 levels of headings (h1 to h6) to create a content hierarchy.

🔠 Headings improve SEO and accessibility: <h1> is the most important (usually one per page), while <h6> is the least. Use <strong> for bold meaning and <em> for italics.

<h1>Main Title</h1>
<p>This is a <strong>bold</strong> word.</p>

Chapter 05

Hyperlinks (Anchors)

The <a> tag allows users to navigate between pages using the href attribute.

🔗 Use target="_blank" to open links in a new tab. Absolute links go to external URLs, relative links navigate within your site.

<a href="https://google.com" target="_blank">Open Google</a>

Chapter 10

Div vs Span

Div is a block level container (takes full width). Span is inline (takes only content width).

🎨 Use div for layout wrappers and span for small inline styling like highlighting words.

I am a div - I fill the space.

I contain a span element inside a sentence.

Chapter 15

Form Validation

Use attributes like required, min, and max to validate data automatically.

✅ Built-in validation prevents empty or invalid submissions. Combine with patterns (pattern="[A-Za-z]{3}") for custom rules.

<input type="email" required>
Chapter 16

Industry Best Practices

The Do's
  • Use Semantic Tags
  • Include alt for images
  • Keep tags lowercase
The Don'ts
  • Don't use Tables for layout
  • Don't skip the <!DOCTYPE>
  • Avoid inline styles

✨ Validate your HTML with W3C Validator and always write accessible code. This makes your site future-proof and inclusive.
