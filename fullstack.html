🔐 Authentication Flow (JWT / Session)

User → Login → Frontend → POST /api/login → Backend → verify DB → JWT token → Frontend stores → Authorized requests 
1. Client sends credentials
2. Server validates, creates JWT
3. Client stores token (localStorage)
4. Client sends token in Authorization header
5. Server verifies signature → protected route
1. Login → server creates session ID
2. Set cookie (HttpOnly)
3. Browser auto-sends cookie on each request
4. Server validates session from store

🔄 API Request Lifecycle (REST)

  1. Client → HTTP request (GET /api/users)
  2. Backend route handler → validate params
  3. Service layer → business logic
  4. Database query → return result set
  5. Backend formats JSON → HTTP response
  6. Client parses JSON → updates UI
// Example: Express + MongoDB
app.get('/api/users', async (req, res) => {
  const users = await User.find();
  res.json(users);
});

🛡️ Security Best Practices

  • ✅ HTTPS everywhere, HSTS headers
  • ✅ Hash passwords (bcrypt, PBKDF2)
  • ✅ CSRF protection (tokens)
  • ✅ XSS sanitization (escape output)
  • ✅ Rate limiting & input validation
  • ✅ Environment variables (never commit secrets)
