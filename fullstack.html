📖 1. What is Full Stack Development?

Full Stack Development is building both the frontend (user interface) and backend (server, logic, database) of a web application. A full stack developer can create a complete product from scratch.
🍽️ Analogy: Frontend = menu & tables; Backend = kitchen; Database = fridge; API = waiter.

💡 2. Why is Full Stack Development Important?

  • ✔ End-to-end understanding
  • ✔ Faster development cycles
  • ✔ Easier troubleshooting
  • ✔ Scalability
  • ✔ Higher employability & freelancing

🔗 6. How They Connect (HTTP & APIs)

The frontend communicates with the backend via HTTP requests (GET, POST, PUT, DELETE). The backend exposes API endpoints that return JSON data.
// Frontend fetch
fetch('/api/products')
  .then(res => res.json())
  .then(data => console.log(data));

// Backend response (JSON)
{ "id": 1, "name": "AI Service", "price": 99 }
💡 The browser never directly accesses the database – it always goes through the backend API.

🔐 7. Authentication Flow (JWT)

  1. User submits login credentials → frontend POST /api/login
  2. Backend validates user against database
  3. Backend generates signed JWT token
  4. Frontend stores token (localStorage or cookie)
  5. Subsequent requests include Authorization: Bearer <token>
  6. Backend verifies token before sending protected data
// JWT verification middleware
function auth(req, res, next) {
  const token = req.header('Authorization')?.replace('Bearer ', '');
  if (!token) return res.status(401).json({ error: 'Access denied' });
  try { req.user = jwt.verify(token, SECRET); next(); } 
  catch { res.status(400).json({ error: 'Invalid token' }); }
}

🔄 8. API Request Lifecycle (REST)

1️⃣ Client → HTTP request (GET /api/products)
2️⃣ Backend router → controller → validate input
3️⃣ Service / business logic
4️⃣ Database query (SELECT * FROM products)
5️⃣ Database returns result → backend formats JSON
6️⃣ Backend sends HTTP response (status 200, 404, etc.)
7️⃣ Frontend parses JSON and updates UI

☁️ 9. Deployment & Hosting

Modern pipeline: GitHub → CI/CD (GitHub Actions) → Build → Deploy

Frontend: Vercel, Netlify
Backend: Render, Railway, Heroku
Database: MongoDB Atlas, Supabase
Container: Docker, Kubernetes
# Deploy frontend to Vercel
vercel --prod

# Deploy backend with Docker
docker build -t myapp .
docker run -p 3000:3000 myapp

🛡️ 10. Security Best Practices

✅ HTTPS everywhere
✅ Hash passwords (bcrypt)
✅ CSRF tokens
✅ XSS sanitization
✅ Rate limiting
✅ Environment variables
// Never hardcode secrets
const dbPassword = process.env.DB_PASSWORD; // ✅
