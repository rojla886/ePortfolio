🔥 HTML for Beginners: Complete Guide + Live Examples

Welcome! This page teaches you HTML from scratch. Every tag is explained in plain English, shown live, and includes a code snippet. No prior knowledge needed.

🔰 1. What is HTML?

📖 Simple definition: HTML (HyperText Markup Language) is the code used to create web pages. Think of it as the skeleton of a website – it gives structure: headings, paragraphs, images, links, etc.

Live example: This very page is built with HTML. Right‑click anywhere and select "View Page Source" to see the code behind it. 

<!DOCTYPE html>
<html>
  <head>
    <title>My First Page</title>
  </head>
  <body>
    <h1>Hello World!</h1>
    <p>This is my first paragraph.</p>
  </body>
</html>
💡 Remember: HTML uses tags (words inside angle brackets, like <p>). Most tags come in pairs: an opening tag <p> and a closing tag </p>. The content goes between them.

🏗️ 2. Document Structure (The Skeleton)

2.1 <!DOCTYPE html>

This tells the browser: "I am using HTML5 (the latest version)". Always put it at the very top.

2.2 <html> ... </html>

The root element that wraps all the content on the page. All other tags go inside it.

2.3 <head> ... </head>

Contains meta-information (things the user doesn’t see directly): page title, character set, CSS links, etc.

2.4 <body> ... </body>

Contains everything that appears on the screen: text, images, buttons, forms – everything visible.

✅ This entire page uses those tags. Check the browser tab – the title is "HTML for Beginners – Full Documentation + Live Demos".

📝 3. Headings & Text Formatting

3.1 Headings (<h1> to <h6>)

Headings define titles and subtitles. <h1> is the most important (usually the page title), <h2> is less important, down to <h6>. Search engines use headings to understand your content.

3.2 Paragraphs (<p>)

The <p> tag defines a paragraph. Browsers automatically add some space before and after paragraphs.

This is a paragraph. It contains a block of text. You can write several sentences here, and the browser will wrap them appropriately.

This is another paragraph. There is a gap between paragraphs.

3.3 Inline Text Formatting

These tags change the appearance or meaning of text inside a paragraph or heading.

<strong> makes text bold and important (screen readers emphasise it).

<em> makes text italic and emphasised.

<mark> highlights text like a yellow marker.

<small> makes text smaller (for fine print).

<del> shows deleted text (strikethrough).

<ins> shows inserted text (underlined).

H2O (subscript with <sub>)

E=mc2 (superscript with <sup>)

<code> displays inline code like a programming keyword.

Press Ctrl+S to save (keyboard input).

<p><strong>Bold</strong> and <em>italic</em>.</p>
<p>H<sub>2</sub>O and x<sup>2</sup></p>
<p><code>alert('Hello');</code></p>
✅ Best practice: Use <strong> instead of <b>, and <em> instead of <i> – they add meaning, not just style.

🔗 5. Links (<a> – The Anchor Tag)

Links are what make the web "hypertext". They allow users to navigate between pages or to different sections of the same page. The href attribute specifies the destination.

External link: Go to Google (opens new tab)

Internal link: Jump to the Tables section below (scrolls to element with id="tables")

Email link: Send an email

Phone link: Call us 

<a href="https://example.com" target="_blank">Visit Example</a>
<a href="#section2">Go to Section 2</a>
<a href="mailto:hello@example.com">Email Us</a>
💡 target="_blank" opens the link in a new tab. Always use it for external websites so users don't leave your page.

✅ 11. HTML Best Practices for Beginners

  • Always start with <!DOCTYPE html>.
  • Use semantic tags (<header>, <nav>, <main>) instead of too many <div>s.
  • Add alt attributes to every image for accessibility.
  • Keep a logical heading order: <h1> then <h2> (don’t skip levels).
  • Close all tags (except self‑closing like <img> and <br>).
  • Use lowercase for tag names and attributes.
  • Validate your HTML with the W3C Validator.
