Chapter 01

What is Full Stack Development?

Full stack development is the practice of building both the front end (client side) and back end (server side) of a web application, along with managing the database. A full stack developer can handle:
  • User interfaces with HTML, CSS, JavaScript, and modern frameworks (React, Vue, Angular)
  • Server logic using Node.js, Python (Django/Flask), PHP, Ruby on Rails, or Java
  • Databases – SQL (PostgreSQL, MySQL) or NoSQL (MongoDB)
  • APIs – RESTful or GraphQL to connect frontend and backend
  • Deployment & DevOps – cloud platforms (AWS, Vercel, Heroku), Docker, CI/CD
🍽️ Restaurant Analogy:
• Frontend = menu, tables, ambience (customer experience).
• Backend = kitchen (prepares food, handles orders).
• Database = fridge/pantry (stores ingredients).
• API = waiter (takes order to kitchen, brings food back).

Chapter 05

Authentication Flow (JWT)

JWT (JSON Web Token) is a secure way to authenticate users. After login, the backend issues a signed token. The frontend stores it (localStorage) and sends it with every subsequent request. 
// JWT verification middleware (Node.js)
function authenticateToken(req, res, next) {
  const token = req.headers['authorization']?.split(' ')[1];
  if (!token) return res.sendStatus(401);
  jwt.verify(token, process.env.JWT_SECRET, (err, user) => {
    if (err) return res.sendStatus(403);
    req.user = user;
    next();
  });
}

Chapter 08

Security Best Practices

  • HTTPS & HSTS – encrypt all traffic
  • Hash passwords with bcrypt (never plain text)
  • CSRF protection & XSS sanitisation
  • Environment variables for secrets
  • Rate limiting to prevent brute force
  • Input validation & parameterised queries
// Password hashing example (Node.js with bcrypt)
const bcrypt = require('bcrypt');
const saltRounds = 10;
const hashedPassword = await bcrypt.hash(password, saltRounds);
// Store hashedPassword in database, never the plain password
