Chapter 01

What is Full Stack Development?

Full stack development is the practice of building both the front end (client side) and back end (server side) of a web application, along with managing the database. A full stack developer can handle:
  • 🎨 User interfaces with HTML, CSS, JavaScript, and modern frameworks (React, Vue, Angular)
  • ⚙️ Server logic using Node.js, Python (Django/Flask), PHP, Ruby on Rails, or Java
  • 🗄️ Databases – SQL (PostgreSQL, MySQL) or NoSQL (MongoDB)
  • 🔌 APIs – RESTful or GraphQL to connect frontend and backend
  • ☁️ Deployment & DevOps – cloud platforms (AWS, Vercel, Heroku), Docker, CI/CD
🍽️ Restaurant Analogy:
• Frontend = menu, tables, ambience (customer experience).
• Backend = kitchen (prepares food, handles orders).
• Database = fridge/pantry (stores ingredients).
• API = waiter (takes order to kitchen, brings food back).

Chapter 04

How It Works: API Request Lifecycle

  1. Frontend sends HTTP request (GET /api/products)
  2. Backend route handler validates request, extracts data
  3. Service/Controller executes business logic
  4. Database query – SQL or NoSQL operation
  5. Backend formats JSON response and sends HTTP status
  6. Frontend receives response and updates UI dynamically

Chapter 05

Authentication Flow (JWT)

  1. User submits login credentials
  2. Backend validates against database
  3. Backend generates signed JWT token
  4. Frontend stores token (localStorage / cookies)
  5. Subsequent requests include token in Authorization: Bearer <token>
  6. Backend verifies token before sending protected data
// JWT verification middleware
function authenticateToken(req, res, next) {
  const token = req.headers['authorization']?.split(' ')[1];
  if (!token) return res.sendStatus(401);
  jwt.verify(token, process.env.JWT_SECRET, (err, user) => {
    if (err) return res.sendStatus(403);
    req.user = user;
    next();
  });
}

Chapter 08

Security Best Practices

  • 🔒 HTTPS & HSTS – encrypt all traffic
  • 🔑 Hash passwords with bcrypt (never plain text)
  • 🛡️ CSRF protection & XSS sanitisation
  • 📦 Environment variables for secrets
  • ⏱️ Rate limiting to prevent brute force
  • ✅ Input validation & parameterised queries
💡 Never commit .env files or API keys. Use .gitignore and secret managers.
