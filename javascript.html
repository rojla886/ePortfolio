📚 Detailed Explanations of JavaScript Concepts

Below, each concept used in this page is explained with a definition and a concrete code example from the actual demo.

1. Variables & Data Types (let, const, arrays, objects, booleans)

Definition: Variables store data. let allows reassignment, const is for values that won't change. Data types include arrays (lists), objects (key‑value pairs), booleans (true/false). 
// Array of task objects
let tasks = [
    { id: 1, text: "Learn JS", completed: false }
];
// Boolean variable for filter state
let currentFilter = 'all'; // string
// Const for DOM element (never changes)
const taskInput = document.getElementById('taskInput');
💡 From our code: tasks is an array of objects. currentFilter stores the active filter ('all', 'active', 'completed'). const is used for DOM references because they don't change.

2. Functions (addTask, renderTasks, saveToLocalStorage, validateForm)

Definition: Functions are reusable blocks of code. They can take parameters and return values. They help organise code and avoid repetition. 
// Function to add a new task
function addTask() {
    const text = taskInput.value.trim();
    if (text) {
        tasks.push({ id: Date.now(), text, completed: false });
        saveTasks();      // call another function
        renderTasks();    // update UI
        taskInput.value = '';
    }
}
// Function to save tasks to localStorage
function saveTasks() {
    localStorage.setItem('js_tasks', JSON.stringify(tasks));
}
💡 From our code: addTask() collects input, updates the array, saves to storage, and refreshes the display – all in one function.

3. Event Listeners (click, input, keypress, submit)

Definition: Event listeners wait for user actions (clicks, typing, etc.) and run a function in response. 
// Click event on Add button
document.getElementById('addTaskBtn').addEventListener('click', addTask);
// Keypress event on input field (press Enter)
taskInput.addEventListener('keypress', (e) => {
    if (e.key === 'Enter') addTask();
});
// Input event for real‑time validation
emailInput.addEventListener('input', validateEmail);
💡 From our code: The validation updates as you type (input event). The to‑do list adds a task when you press Enter (keypress).

4. DOM Manipulation (createElement, appendChild, innerHTML, classList)

Definition: The DOM (Document Object Model) is the browser's representation of the HTML. JavaScript can create, modify, or remove elements. 
// Create a new task element
const taskDiv = document.createElement('div');
taskDiv.className = 'task-item';
const textSpan = document.createElement('span');
textSpan.textContent = task.text;
textSpan.classList.add('task-text');
if (task.completed) {
    textSpan.classList.add('completed');
}
taskDiv.appendChild(textSpan);
container.appendChild(taskDiv);
💡 From our code: Each task is dynamically created using createElement, styled with classList, and inserted with appendChild.

5. Array Methods (push, filter, map, forEach, splice)

Definition: Arrays have built‑in methods to manipulate data. push adds, filter selects items, forEach loops, splice removes. 
// Add new task
tasks.push(newTask);
// Filter tasks based on completion status
let filtered = tasks;
if (currentFilter === 'active') {
    filtered = tasks.filter(t => !t.completed);
}
// Loop through tasks to render
filtered.forEach(task => {
    // render each task
});
// Delete task by removing from array
tasks = tasks.filter(t => t.id !== taskId);
💡 From our code: filter is used for the All/Pending/Completed buttons. forEach loops through tasks to display them.

6. LocalStorage (setItem, getItem, JSON.stringify/parse)

Definition: localStorage saves key‑value pairs in the browser. Data persists after page refresh. Since localStorage only stores strings, we use JSON.stringify to convert arrays/objects to strings, and JSON.parse to convert back. 
// Save tasks
localStorage.setItem('js_tasks', JSON.stringify(tasks));
// Load tasks
const saved = localStorage.getItem('js_tasks');
if (saved) {
    tasks = JSON.parse(saved);
}
💡 From our code: Your tasks remain even after refreshing the page – try it! This simulates a real database.

7. Conditional Logic (if/else, ternary operators)

Definition: Conditionals control which code runs based on a condition. if/else is the classic form; the ternary operator (? :) is a shorter way. 
// if/else for validation
if (password.length >= 6) {
    passwordError.textContent = '';
    isValid = true;
} else {
    passwordError.textContent = 'Password too short';
    isValid = false;
}
// Ternary operator for CSS class
textSpan.classList.add(task.completed ? 'completed' : '');
💡 From our code: The validation functions use if/else. The task text gets the 'completed' class conditionally using a ternary operator.

8. Template Literals (`string ${variable}`)

Definition: Template literals (backticks) allow embedding variables directly into strings using ${...}. They can also span multiple lines. 
// Display stats message
statsDiv.innerHTML = `${completedCount} of ${total} tasks completed (${Math.round(percent)}%)`;
// Dynamic HTML creation (not used here but common)
const html = `
${taskText}
`;
💡 From our code: The task counter uses template literals to insert numbers into a sentence. Easier than string concatenation.
