📚 JavaScript Concepts – Explained with Code

1. Variables: let, const, var

let – value can be changed. const – value cannot be reassigned (used for DOM elements, arrays/objects can still be mutated). var – old way (avoid). Variables store data: strings, numbers, booleans, arrays, objects. 
let tasks = [];           // can be reassigned
const taskInput = document.getElementById('taskInput'); // never changes
let count = 0;            // number
const isCompleted = false; // boolean

2. Functions (Reusable code blocks)

Functions group code that performs a specific task. They can take parameters and return values. Used to avoid repetition. 
function addTask() {
  const text = taskInput.value.trim();
  if (text) {
    tasks.push({ id: Date.now(), text, completed: false });
    saveTasks();    // call another function
    renderTasks();
  }
}

3. Event Listeners (click, input, keypress, submit)

Event listeners wait for user actions and run a function. Common events: click, input, keypress, submit. 
addBtn.addEventListener('click', addTask);
taskInput.addEventListener('keypress', (e) => {
  if (e.key === 'Enter') addTask();
});
emailInput.addEventListener('input', validateEmail);

4. DOM Manipulation (createElement, appendChild, classList)

The DOM is the browser's representation of the HTML. JavaScript can add, remove, or modify elements dynamically. 
const taskDiv = document.createElement('div');
taskDiv.className = 'task-item';
const textSpan = document.createElement('span');
textSpan.textContent = task.text;
textSpan.classList.add('task-text');
if (task.completed) textSpan.classList.add('completed');
taskDiv.appendChild(textSpan);
container.appendChild(taskDiv);

5. Array Methods (push, filter, map, forEach, splice)

Arrays have built‑in methods to manipulate data: push (add), filter (select), forEach (loop), map (transform), splice (remove). 
// Add
tasks.push(newTask);
// Filter by completion
const activeTasks = tasks.filter(t => !t.completed);
// Loop to render
filteredTasks.forEach(task => { /* render */ });
// Delete
tasks = tasks.filter(t => t.id !== taskId);

6. LocalStorage (persist data after refresh)

localStorage saves key‑value pairs in the browser. Data persists after page reload. Use JSON.stringify to convert objects/arrays to strings, and JSON.parse to convert back. 
// Save
localStorage.setItem('js_tasks', JSON.stringify(tasks));
// Load
const saved = localStorage.getItem('js_tasks');
if (saved) tasks = JSON.parse(saved);
💡 Your to‑do list tasks remain even after refreshing – try it!

7. Promises & Async/Await (Handling asynchronous operations)

JavaScript often needs to wait for things (API calls, timers). Promises represent a future value. Async/Await makes promise code easier to read. The Fetch API is used to request data from servers. 
// Async function to fetch data
async function fetchUser() {
  try {
    const response = await fetch('https://jsonplaceholder.typicode.com/users/1');
    if (!response.ok) throw new Error('Network error');
    const user = await response.json();
    console.log(user.name);
  } catch (error) {
    console.error('Failed:', error);
  }
}
💡 Click the button – it fetches a random user from a public API asynchronously, without reloading the page.

8. Error Handling (try...catch)

try...catch prevents your script from crashing when something goes wrong (e.g., network failure, invalid data). The catch block handles the error gracefully. 
try {
  // Code that might fail
  const data = JSON.parse(invalidJSON);
} catch (error) {
  console.error('Caught error:', error.message);
  // Show user‑friendly message
  errorDiv.textContent = 'Something went wrong. Please try again.';
}

9. Classes (Object‑oriented programming)

Classes are blueprints for creating objects. They encapsulate data and methods. Modern JavaScript uses classes for components and models. 
class Task {
  constructor(text) {
    this.id = Date.now();
    this.text = text;
    this.completed = false;
  }
  toggle() {
    this.completed = !this.completed;
  }
}
const myTask = new Task('Learn classes');
myTask.toggle();
console.log(myTask.completed); // true

10. Template Literals (Backticks `${variable}`)

Template literals allow embedding variables and expressions inside strings using ${...}. They also support multi‑line strings. 
const completed = 5;
const total = 10;
const message = `${completed} of ${total} tasks done (${(completed/total)*100}%)`;
console.log(message);
