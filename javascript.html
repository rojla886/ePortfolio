🧠 What is JavaScript?

JavaScript (JS) is the programming language that makes websites interactive and dynamic. While HTML gives structure and CSS provides styling, JavaScript brings behaviour – handling button clicks, validating forms, updating content without reloading, fetching data from servers, creating animations, and much more.

Created in 1995 by Brendan Eich in just 10 days, JavaScript has evolved into one of the most popular programming languages in the world. Today, it runs on every modern web browser (Chrome, Firefox, Safari, Edge) and also on servers using Node.js, allowing developers to build complete applications (frontend + backend) with a single language.

Why learn JavaScript? It is essential for front‑end development, enables full‑stack development (with Node.js), has a huge ecosystem of libraries and frameworks (React, Vue, Angular), and offers excellent job opportunities.

Click the button below – JavaScript changes the text and style instantly!

// Basic example: change content and style with JavaScript
document.getElementById('welcomeBtn').addEventListener('click', () => {
  const msg = document.getElementById('welcomeMessage');
  msg.textContent = 'Hello! JavaScript is working 🎉';
  msg.style.color = '#2c7da0';
  msg.style.fontWeight = 'bold';
});
💡 JavaScript runs in the browser's JavaScript engine (e.g., Chrome's V8). It is an interpreted language – no compilation needed.

📚 JavaScript Concepts – Explained with Code

1. Variables: let, const, and var

Variables are containers for storing data. In JavaScript, we declare variables using:
  • let – for values that can change later. Example: let age = 25; age = 26;
  • const – for values that should never be reassigned. Used for DOM elements, configuration values. Note: objects/arrays declared with const can still be modified (e.g., push to array), but you cannot reassign the variable itself.
  • var – the old way (pre‑ES6). Avoid it because it has function scope and can lead to bugs.
Data types include: strings ("hello"), numbers (42), booleans (true/false), arrays ([1,2,3]), objects ({name:"John"}), and more. 
let tasks = [];           // can be reassigned later
const taskInput = document.getElementById('taskInput'); // never changes
let count = 0;            // number
const isCompleted = false; // boolean
💡 In our to‑do list, tasks is a let variable because we reassign it when deleting or adding tasks. taskInput is a const because it always refers to the same DOM element.

2. Functions (Reusable code blocks)

Functions are reusable blocks of code that perform a specific task. They help avoid repetition, organise code, and make it testable. A function can take parameters (inputs) and optionally return a value.

You define a function with the function keyword (or using arrow functions () => {}). Then you call it by its name followed by parentheses. 
// Function to add a new task (from our to‑do list)
function addTask() {
  const text = taskInput.value.trim();
  if (text !== '') {
    tasks.push({ id: Date.now(), text, completed: false });
    saveTasks();    // call another function
    renderTasks();  // update the UI
    taskInput.value = '';
  }
}

// Arrow function example
const greet = (name) => `Hello ${name}!`;

3. Event Listeners (click, input, keypress, submit)

Event listeners wait for user actions (clicks, typing, mouse movements, form submission) and then execute a function. This is what makes web pages interactive.

Common events: click (button pressed), input (user types in a field), keypress (keyboard key), submit (form sent). You attach an event listener using addEventListener. 
// Click event on Add button
addBtn.addEventListener('click', addTask);

// Keypress event – press Enter in the input field
taskInput.addEventListener('keypress', (e) => {
  if (e.key === 'Enter') addTask();
});

// Input event for real‑time validation
emailInput.addEventListener('input', validateEmail);

4. DOM Manipulation (createElement, appendChild, classList)

The DOM (Document Object Model) is the browser's representation of the HTML page as a tree of objects. JavaScript can manipulate this tree – creating, removing, or modifying elements dynamically, without reloading the page.

Key methods: document.getElementById(), document.querySelector(), createElement(), appendChild(), classList.add()/remove(). 
// Create a new task element (from our to‑do list)
const taskDiv = document.createElement('div');
taskDiv.className = 'task-item';
const textSpan = document.createElement('span');
textSpan.textContent = task.text;
textSpan.classList.add('task-text');
if (task.completed) textSpan.classList.add('completed');
taskDiv.appendChild(textSpan);
container.appendChild(taskDiv);

5. Array Methods (push, filter, map, forEach, splice)

Arrays are lists of data. JavaScript provides powerful built‑in methods to manipulate them:
  • push() – add an item to the end
  • filter() – create a new array with items that pass a test
  • forEach() – execute a function for each array element
  • map() – transform each element into a new value
  • splice() – remove or replace items
These methods are immutable where appropriate and make array handling concise. 
// Add new task
tasks.push(newTask);

// Filter active tasks (not completed)
const activeTasks = tasks.filter(task => !task.completed);

6. LocalStorage (persist data after page refresh)

localStorage is a web storage API that allows you to store key‑value pairs in the browser. Data persists even after the page is closed and reopened (unlike sessionStorage which expires). It's perfect for saving user preferences, to‑do lists, or authentication tokens.

localStorage only stores strings. To store objects or arrays, use JSON.stringify() to convert them to a string, and JSON.parse() to convert back. 
// Save tasks to localStorage
localStorage.setItem('js_tasks', JSON.stringify(tasks));

// Load tasks from localStorage
const saved = localStorage.getItem('js_tasks');
if (saved) {
  tasks = JSON.parse(saved); // convert back to array
}
💡 In our to‑do list, tasks are saved automatically. Try adding a task, refresh the page – it's still there! This simulates a real database.

7. Promises & Async/Await (Handling asynchronous operations)

JavaScript is asynchronous by default – it doesn't wait for slow operations (like network requests) to finish before moving on. Promises represent a future value (pending, resolved, or rejected). Async/await is a modern syntax that makes working with promises easier and more readable.

The Fetch API is used to make HTTP requests to servers (e.g., get data from an API). It returns a Promise. 
// Async function to fetch user data
async function fetchUser() {
  try {
    const response = await fetch('https://jsonplaceholder.typicode.com/users/1');
    if (!response.ok) throw new Error('Network error');
    const user = await response.json();
    console.log(user.name);
  } catch (error) {
    console.error('Failed:', error);
  }
}

8. Error Handling (try...catch)

try...catch allows you to handle errors gracefully without crashing the entire script. The try block contains code that might throw an error; if an error occurs, the catch block runs. This is essential for network requests, user input parsing, and any operation that might fail. 
try {
  // Intentionally cause an error (invalid JSON)
  const invalidJSON = '{ name: "John" }'; // missing quotes
  const data = JSON.parse(invalidJSON);
} catch (error) {
  console.error('Caught error:', error.message);
  errorDiv.textContent = 'Something went wrong. Please try again.';
}

9. Classes (Object‑oriented programming)

Classes are blueprints for creating objects with similar properties and methods. They are a core part of object‑oriented programming (OOP). A class can have a constructor to initialise properties, and methods to define behaviour. Classes make code more organised and reusable. 
class Task {
  constructor(text) {
    this.id = Date.now();
    this.text = text;
    this.completed = false;
  }
  toggle() {
    this.completed = !this.completed;
  }
}
const myTask = new Task('Learn classes');
myTask.toggle();
console.log(myTask.completed); // true

10. Template Literals (Backticks and ${})

Template literals (enclosed in backticks `` ` ``) allow embedding variables and expressions directly inside strings using ${...}. They also support multi‑line strings without concatenation. This is much cleaner than the old 'string' + variable approach. 
const completed = 5;
const total = 10;
const message = `${completed} of ${total} tasks done (${(completed/total)*100}%)`;
console.log(message); // "5 of 10 tasks done (50%)"
